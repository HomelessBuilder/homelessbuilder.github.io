The System

What a coordinated agent fleet actually looks like

orchestrator: tracks state, advises, onboards; executes nothing
client site paid product course platform journal +11 more
shared activity log · read-only cron monitor

Each project runs its own scoped agent instance governed by a context file. A separate monitor observes the whole ecosystem with zero write permissions.

  • Staged automation as a governing rule. Every workflow moves manual → prompted → drafted → autonomous, and never skips a stage. Guardrails are designed in before capability is turned up.
  • Observability without authority. A cron-driven monitor reads git logs and key files across every repo and reports on ecosystem health: it can watch everything and change nothing.
  • Documentation as a product. Every project carries its vision and context in version-controlled files an agent (or a new hire) can onboard from. The practice went from first commit to a sellable product in two weeks because it was written down as it was built.
